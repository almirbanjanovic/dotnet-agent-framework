## Task
Reconcile label convention between NetworkPolicy pod selectors and Helm chart templates to resolve Finding 1 (HIGH severity) from Cleveland's security review.

## Problem
- NetworkPolicy files used `app: {service-name}` selector
- Helm templates generate `app.kubernetes.io/name: {service-name}` (Kubernetes standard)
- Result: NetworkPolicy allow rules never match Helm-deployed pods under default-deny baseline
- Impact: Complete service outage on AKS

## Solution (Option A — Selected)
Update all 8 per-service NetworkPolicy files to use `app.kubernetes.io/name: {service-name}` as pod selectors and matchLabels. This aligns with Kubernetes convention and Helm defaults (zero template changes required).
