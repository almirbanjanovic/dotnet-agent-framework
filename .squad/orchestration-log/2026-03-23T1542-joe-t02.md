## Network Design
- **Ingress:** Each service allows traffic only from documented callers per architecture DAG
- **Egress:** All services allow downstream services on 8080, DNS (UDP/TCP 53), and PE subnet (10.0.3.0/24:443)
- **DNS:** All services allow egress to kube-dns (10.96.0.10)
- **Monitoring:** Deferred — no scraper ingress defined; flagged for future work
- **AGC:** bff-api and blazor-ui allow ingress from azure-alb-system namespace (Low severity, acceptable)

## Initial Label Strategy
- Pod selector: `app: {service-name}` (non-standard, will be reconciled with Helm in Follow-up Task)
- **⚠️ Known Issue:** Helm standard labels are `app.kubernetes.io/name: {service-name}`, not `app`. This mismatch will cause traffic to be blocked under default-deny when deployed via Helm. Cleveland security review flagged this as HIGH severity.

## Implementation Notes
- Port 8080 hardcoded (matches .NET 9 non-root default)
- PE subnet CIDR hardcoded (10.0.3.0/24; if Terraform overrides, NetworkPolicies must update)
- AGC controller assumed in azure-alb-system namespace; if moved, update selectors
